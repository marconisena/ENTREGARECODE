# Agência de Viagens - Atividade Prática do Módulo 5

Este projeto é parte da Atividade Prática do Módulo 5 do Curso RecodePro, onde buscamos aprimorar nossa aplicação web Agência de Viagens, seguindo padrões de projeto e adotando o Ecossistema Spring Framework. O objetivo é refatorar o código, implementando o padrão MVC no back-end com Java Spring Boot, Spring MVC, e persistência de dados utilizando Spring Data JPA. Além disso, exploramos opções para o front-end, incluindo a possibilidade de utilizar a biblioteca React ou qualquer Java template engine, como o Thymeleaf.

## Tecnologias Utilizadas

- *Back-end:*
  - Java Spring Boot
  - Persistência de Dados com Spring Data JPA
  É necessário abri-lo no spring e conectar com o banco de dados ao executar a API.
Para testar as funções acesse no navegador: http://localhost:8080/swagger-ui/index.html
É possivel fazer o CRUD e conferir no MySQL no banco criado chamado "bflorturismo-api".
  - ORM (Object-Relational Mapping)


## Configuração do Banco de Dados

A base de dados é criada ou atualizada seguindo o padrão NoSQL (MongoDB) ou SQL relacional, de acordo com as especificações do projeto. Utilizamos o framework Spring Data para facilitar a integração e aplicamos ORM para mapeamento objeto-relacional.

## Refatoração do Back-end

O código foi refatorado seguindo os padrões de projeto MVC para organizar a aplicação de maneira eficiente. Utilizamos os frameworks Spring para atingir esse objetivo, garantindo a manutenibilidade e escalabilidade do código.


## Instruções para Execução

1. Clone o repositório.
2. Configurar o banco de dados conforme as especificações do projeto.
3. Execute o back-end usando Java Spring Boot.
